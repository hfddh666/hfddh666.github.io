第一章

数据（int,string,bool…)

数据元素：组成数据的基本单位

数据对象：数据元素的集合

数据结构定义: 是一门研究非数值计算的程序设计问题中

计算机的操作对象以及它们之间的关系和操作等等的学科。

索引存储：建立存储的节点外，还建立附加索引表标识节点的地址（B+树）

散列存储结构：就是根据结点的关键字直接计算出该结点 的存储地址

数据类型允许复合嵌套，相同数据结构的同一类型

算法是若干指令的有穷序列，满足性质：

u 输 入：有外部提供的量作为算法的输入。

u 输 出：算法产生至少一个量作为输出。

u 确定性：组成算法的每条指令是清晰，无歧义的。

u 有限性：算法中每条指令的执行次数是有限的，执

行每条指令的时间也是有限的。

时间复杂度计算

题目

解决问题的效率，跟数据的组织方式有关。

抽象数据类型中，描述数据类型的方法与数据存储的物理结构无关。

空间复杂度是根据算法写成的程序在执行时占用存储单元的长度，往往与输入数据的规模有关。

时间复杂度是根据算法写成的程序在执行时耗费时间的长度，往往与输入数据的规模有关。

数据类型由数据对象集和数据集合相关联的操作集组成。

Θ ：描述上下界

Ω ：最好情况

O ：最坏情况

第五章排序

简单排序

1冒泡排序：第一个与第二个比，第二个与第三个比以此类推，T(n)=O(n²)

2直接插入排序：先把第一个看成有序序列，然后按顺序一个一个插入；T(n)=O(n²)

3希尔排序（缩小增量法）：希尔排序的时间复杂度约为O(n 1 . 3 )，原理看上面

4选择排序：依次找出最小的或最大的，T(n)=O(n²)

**快速排序：**快速排序的平均时间复杂度为O (nlog2n)，

为非平衡二分法：选择一个i为分段点，当i为最后一个数时，时间最慢，为n平方

合并排序：

平衡的二分分治

1递归版：

基于比较的排序问题的计算时间下界为Ω(nlogn)

2非递归：

4计数排序：按顺序写下所有值，记下他们的个数，然后输出

上述排序并非稳定排序。

桶排序：

对个位十位百位依次比较也是基数排序

第七章散列表

开散列：空间无穷；

闭散列：空间有限

题目

将M个元素存入用长度为S的数组表示的散列表，则该表的装填因子为M/S

若装填因子α为1，则向散列表中散列元素时一定会产生冲突.因为 对于开散列（链地址法） 来说，即使初始桶被占用，也只是在该桶的链表上增加结点，这依然属于“冲突”

散列表中的同义词是指两个（或更多）不同的关键字，经过散列函数计算后，得到相同的散列地址（桶下标）。

一般再散列的表长是用初始散列的表长的两倍

查找成功概率：长度/现有元素

失败：该位置到空元素的总长度/表长（空元素算1）

正常线性探测删除后元素有标记

第九章并差集

在并查集算法中，若按集合大小执行合并操作，则任一节点的深度不超过 N/2，但并非 O(logN) （错）

如果一棵树是通过按大小合并（union-by-size）由 n 个节点构建而成的，那么在合并过程中，每个元素所属的根节点最多只会改变 log n 次。

假设在不相交集合中，每个单个节点的初始高度为 0，那么通过对 n 个节点执行按高度合并（union-by-height）所形成的最终不相交集合树，其高度将不会超过 ⌊log₂ n⌋。

直接合并也还是要先找到根然后合并

在不相交集合问题中，给定一个包含 m 个元素的集合 S = { 1, 2, 3, …, m } 和 n ( 0 < n < m ) 个不同的等价关系（这里“关系”通常指形如 a 和 b 等价的 Union 操作），则集合 S 一定具有 m-n 个等价类。
